# Issue #4 — test: scaffold playwright with smoke spec

- **Status**: Resolved
- **Milestone**: M1 — Foundation
- **Branch**: `test/4-scaffold-playwright`

## Problem

The agent workflow's mandatory E2E step required a working Playwright setup. None existed.

## Approach

- Installed `@playwright/test` as a devDependency.
- Downloaded the chromium browser via `pnpm exec playwright install chromium`.
- Added `playwright.config.ts`:
  - `testDir: './e2e'`, baseURL `http://127.0.0.1:5173`.
  - `fullyParallel: true`, retries only in CI, `github` + `html` reporters in CI, `list` locally.
  - `webServer` boots `pnpm dev --port=5173 --strictPort` automatically and reuses an existing dev server outside CI.
  - Single chromium project to keep CI fast at this stage; mobile / firefox / webkit can be added later.
- Wrote `e2e/smoke.spec.ts` asserting:
  - The root document has a title.
  - `#root` is not empty (React mounted).
  - No console / pageerror events.
- Added scripts: `e2e` (run), `e2e:ui` (Playwright UI), `e2e:report` (show last report).
- Added Playwright artifact directories to `.gitignore` and ESLint `ignores`.
- Added `playwright.config.ts` to `tsconfig.node.json` `include`.

## Trade-offs

- Only chromium is run in this milestone — sufficient for smoke coverage. Mobile and webkit projects can be added in M3/M4 alongside touch gestures.
- The webServer reuses an existing dev server locally to avoid double-boot during repeated runs; CI always starts a fresh server.
- The smoke spec is broad (no specific UI assertion) since the gallery page lands in issue #6; this spec will be extended there.

## Verification

- `pnpm e2e` → `1 passed (854ms)` (chromium, headless, against `pnpm dev`).
- `pnpm lint` → clean.

### Sandbox note

When running the agent's local shell inside the Cursor sandbox, `PLAYWRIGHT_BROWSERS_PATH` is exported to a temp cache that ended up containing only `mac-arm64` binaries while Playwright's runtime resolved `mac-x64` from the same cache. Running outside the sandbox with the default cache path (`~/Library/Caches/ms-playwright`) works without issue. CI and ordinary developer terminals are unaffected.
