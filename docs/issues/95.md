# Issue #95: Commit target pages after the v3 flip animation

## Problem

The v3 reader used the requested `pageIndex` directly for the static page
layer. Navigation therefore revealed the target page before the outgoing leaf
finished turning.

## Approach

- Separate the requested target index from a visual `displayIndex`.
- Keep the static page layer on `displayIndex` throughout an active turn.
- Commit the target to `displayIndex` only from the progress controller's
  completion callback.
- Guard the callback so a cancelled or superseded turn cannot commit a stale
  target.
- Keep reduced-motion behavior immediate through the controller's existing
  synchronous completion path.

## Trade-offs

- Pagination controls continue to represent the requested target while the
  visual reader and its accessible page label remain on the displayed page
  until the motion completes.
- A future two-sided/WebGL leaf can reveal the incoming page progressively;
  this CSS fallback intentionally prioritizes a clean end-of-turn handoff.
