## Problem

The repository was scaffolded from a generic Vite + React starter (`VRT Starter R1`), so two foundational documents were misaligned with the actual product:

1. There was no machine-readable convention layer for the AI agent. Conventions only existed as prose in `docs/plan-1.md`, which meant the agent could not consistently enforce them across sessions.
2. `README.md` still advertised the upstream starter — wrong title, wrong feature list, stale React Router version (v6 vs v7), stale package manager floor (v9+ vs v10), broken screenshot reference (`public/vrt-260108-1.png`), and no mention of the book-flip-showcase product.

## Approach

### 1. Distill `docs/plan-1.md` into focused Cursor rules

Per `.cursor/rules/` best practices (one concern per rule, < 50 lines, concrete examples), the plan was split into five rules:

| File | Scope | Why split out |
| --- | --- | --- |
| `project-overview.mdc` | `alwaysApply: true` | Persistent project context every session needs. |
| `code-style.mdc` | `src/**/*.{ts,tsx}` | StandardJS rules only matter on TS sources. |
| `commit-convention.mdc` | `alwaysApply: true` | Touches every commit, regardless of file. |
| `agent-workflow.mdc` | `alwaysApply: true` | The mandatory loop the agent must run. |
| `testing.mdc` | test/spec/e2e globs | Vitest/Playwright guidance only when authoring tests. |

Each rule includes BAD/GOOD examples where useful (notably `code-style.mdc`).

### 2. Rewrite `README.md` for `book-flip-showcase`

- Replaced title, badges, and tagline.
- Rebuilt the feature list around the actual product (page-flip reader, thumbnail scrubber, keyboard nav, reduced-motion support).
- Updated the tech stack table to match `package.json`: React Router 7, pnpm 10+, lucide-react, and Vitest/Playwright as planned.
- Added a "Conventions" section that links each `.cursor/rules/*.mdc` file from the README so human contributors can discover the agent rules.
- Removed the broken `vrt-260108-1.png` screenshot and the `-r1` naming convention block (not applicable to this product).
- Added the agent workflow loop and a link to `docs/plan-1.md` as the source of truth.

### 3. Trade-offs

- **Five rule files vs one mega-rule**: Five focused files cost more navigation but match the SKILL.md guidance ("one concern per rule") and let `globs`/`alwaysApply` be tuned per concern. The mega-rule would have been simpler to read once but harder to scope correctly.
- **Retrospective issue**: The original merge happened locally because the `gh` token was stale. Rather than rewriting history, this issue was filed retrospectively and a follow-up PR adds only this report. This preserves the audit trail without destructive rebases.
- **Skipping Vitest/Playwright**: Allowed by `.cursor/rules/agent-workflow.mdc` for pure docs changes. No runtime code was touched.
