# Issue #43 — feat(reader): show the cover alone in spread mode

- **Status**: Resolved
- **Milestone**: M6 — Reader Enhancements
- **Branch**: `feat/43-cover-view`

## Why

Real books open to a single cover on the right. Pairing the cover with page 2 felt like a catalogue, not a book. This issue introduces a **Cover alone** option that, when enabled, lays the cover out on its own in spread mode.

## How

1. **`useCoverMode` hook** persists `'single' | 'spread'` to `localStorage` under `book-flip-showcase:cover-mode`. Default `'single'` (real-book look).
2. **Cover-aware helpers in `useViewMode.ts`**:
   - `snapPage(index, viewMode, coverMode)` — returns the closest valid spread start. With `coverMode='single'`, valid starts are `0, 1, 3, 5, 7, ...`.
   - `effectiveStep(pageIndex, viewMode, coverMode)` — `1` only at the cover, `2` everywhere else in spread mode.
   - `isCoverAlone(pageIndex, viewMode, coverMode)` — boolean predicate.
3. **`PageFlip`** now accepts `coverMode` and `step`. When the current (or outgoing) index lands on a cover-alone page it renders **one** image on the right half and a blank left half, with `data-cover-alone="true"` on the surface.
4. **`useReaderKeyboard`** accepts an optional `snap` function. The `ArrowLeft` guard is relaxed to `pageIndex > 0` so the user can step **back to the cover from the first interior spread**. `End` snaps the last index instead of using step-modulo.
5. **`PageJumpInput`** accepts an optional `snap` function so typed jumps align to valid cover-aware starts.
6. **`ReaderControls`** displays a single page number when the current page is a cover-alone, and shows `"Prev"` enabled iff `pageIndex > 0`.
7. **New `CoverModeToggle` component** — a small switch in the reader header, disabled when the view mode is `single` (the cover-alone concept only applies in spread mode).
8. **Reader** wires everything together. A snap key `${viewMode}:${coverMode}` re-snaps `pageIndex` whenever either preference changes, so the user is never stranded on an invalid index after toggling.

## Acceptance Checks

- [x] Cover alone renders the cover on the right half with a blank `bg-slate-100 dark:bg-slate-950/40` left half.
- [x] Forward from cover advances by 1 (cover → first interior spread, index 1).
- [x] Forward from spread advances by 2 (interior spread → next interior spread).
- [x] Backward from index 1 returns to the cover (index 0).
- [x] `Cover alone` toggle is disabled when view mode is `single`.
- [x] Pagination indicator displays `"1 / N"` at the cover and `"2–3 / N"` at the first interior spread (no longer "1–2").
- [x] `coverMode === 'spread'` preserves the previous catalogue-style pairing — confirmed by the existing spread tests, now seeded with `seedCoverMode(page, 'spread')`.

## Trade-offs

- **Default = single (cover alone)**: matches the real-book convention requested by the user. The "catalogue" look is still one toggle away.
- **`coverMode='spread'` was the previous default**; existing E2E specs that relied on it now seed it explicitly via `seedCoverMode(page, 'spread')`, making the assertion intent explicit rather than implicit.
- **Disabled toggle in single mode** — the cover-alone concept has no meaning when there's only one page on screen, so the control communicates that visually rather than silently no-op-ing.
