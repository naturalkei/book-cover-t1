# #83 — v3 progress-based flip engine and CSS curl fallback

## Problem

v3 shell (#82) exposed a reader stub without a flip renderer. plan-3 requires a progress-driven engine with a CSS segmented curl fallback before WebGL lands in #85.

## Approach

1. Added `flip-progress.ts` with rAF-based `animateFlipProgress` using shared `easePaperProgress`.
2. Implemented `CssCurlLeaf` — 12 vertical segments driven by `sampleCurl` from `curl-model.ts`.
3. Built `PageFlipEngine` orchestrating static layer + outgoing curl leaf with `data-flip-progress`.
4. Wired v3 `Reader` with view/cover toggles, keyboard nav, and pagination controls.
5. Forked v3 hooks (`view-mode`, `cover-mode`, `reader-keyboard`, `reduced-motion`) with v3 storage keys.

v1/v2 untouched.

## Trade-offs

- Spread mode uses half-width curl without phantom half yet (simpler than v2 spread leaf).
- WebGL renderer deferred to #85; `data-flip-renderer="css"` fixed for now.
- Gutter progress lighting deferred to #84.
