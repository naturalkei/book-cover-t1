# Issue #12 — feat(reader): add draggable thumbnail scrubber

- **Status**: Resolved
- **Milestone**: M3 — Advanced Navigation
- **Branch**: `feat/12-thumbnail-scrubber`

## Problem

Long books were tedious to scrub through. The reader needed a slider + thumbnail strip so users could rapidly preview pages and commit a jump on release.

## Approach

- Added `src/components/reader/ThumbnailScrubber.tsx`:
  - Horizontal `<input type="range">` slider (`role="slider"`) with full ARIA value text. Mouse / touch drag updates a local `previewIndex`; commit happens on `pointerup` (and `blur` if focus leaves mid-drag). Keyboard interactions commit immediately, so ArrowLeft / ArrowRight on the slider behave like the existing reader shortcuts.
  - Live preview hint (`data-testid="scrubber-preview"`) shows `preview · page N` while dragging, with `aria-live="polite"`.
  - A `<ul aria-label="Page thumbnails">` row of square thumbnail buttons (`aria-label="Jump to page N"`, `aria-current="page"` on the active one). Click commits the jump.
  - Virtualization: books with > 50 pages only render thumbnails within `pageIndex ± thumbnailWindow` (default 8). Shorter books render all thumbnails since the browser's image lazy-loading already handles off-screen cost.
- Wired the scrubber into `Reader.tsx` between the page-jump input and the footer; it consumes the same `setPageIndex` callback so the flip animation always fires.

## Trade-offs

- **`<input type="range">` vs custom drag**: gives keyboard, touch, and assistive tech support for free, and produces a real `role="slider"` for Playwright. The native thumb is themed via `accent-color` so it fits the dark palette.
- **Drag-preview state vs ref**: lint rule `react-hooks/refs` forbids reading refs during render; the `dragging` flag therefore lives in `useState`, costing one re-render on pointer down / up — negligible at this scale.
- **Virtualization is window-based**: a real virtualized list (e.g., `react-virtual`) would be more efficient for huge books, but the dependency cost is unjustified at three sample books with 8–12 pages. The window approach keeps the DOM small while still demoing the technique.
- **Click jump on thumbnail = no flip animation**: the underlying flip ignores click-while-animating; rapid thumbnail clicks therefore intentionally queue at most one transition. Acceptable for the M3 scope.
