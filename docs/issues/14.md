# Issue #14 — feat(a11y): respect prefers-reduced-motion and pass aria audit

- **Status**: Resolved
- **Milestone**: M4 — Polish
- **Branch**: `feat/14-a11y-reduced-motion`

## Problem

The app needed a real accessibility pass: confirmable WCAG color contrast, a skip link, a focused `<main>` target, an automated audit, and a reduced-motion regression test. The reduced-motion fallback existed in code (introduced in #8) but had no end-to-end coverage.

## Approach

- Added `@axe-core/playwright` and `e2e/a11y.spec.ts`:
  - Loops over `light` and `dark` themes (seeded via `localStorage` before navigation) and audits the gallery and reader pages against WCAG 2.0 / 2.1 A + AA tags.
  - A standalone spec verifies that **Tab** focuses the new skip link and **click** moves focus into `#main-content`.
  - Full keyboard journey (ArrowRight, Home, End, Escape) on the reader.
  - Reduced-motion spec creates a browser context with `reducedMotion: 'reduce'`, advances a page, and asserts the outgoing leaf uses an `opacity` transition (not `transform`).
- Added a "Skip to main content" link as the first focusable element in `Layout`, with a visible focus style and an `id="main-content"` + `tabIndex={-1}` target on `<main>` so the link can route focus.
- Re-labelled the primary `<nav>` with `aria-label="Primary"` to distinguish it from the in-page `Reader pagination` nav.
- **Color contrast pass**: bumped muted text from `slate-500` to `slate-700` (light) / `slate-300` or `slate-400` (dark) in:
  - Gallery hero kicker.
  - `BookCard` page count chip — replaced the per-book translucent accent tint with a neutral `slate-100` / `white/10` chip + `slate-700` / `slate-200` text so the chip always passes AA.
  - `Reader` author byline, footer hint.
  - `ReaderControls` indicator, page-jump label / total separator, scrubber labels and preview text.
  - `Layout` footer.
- All changes verified to keep axe quiet on both themes.

## Trade-offs

- **Dropping per-book accent chip**: the colored chip looked great but mixed a 10% accent over multiple backgrounds, which axe rightly flagged as too low contrast for small text. The neutral chip preserves typography and keeps the per-book accent visible through the card shadow.
- **Tab order**: with the skip link as the first focusable element, every page now needs one extra Tab before reaching the brand link. Acceptable cost for a navigation feature that materially helps keyboard users.
- **Axe filter set**: limited to WCAG A/AA tags so the suite stays actionable; experimental "best-practice" rules are deliberately excluded for now.
