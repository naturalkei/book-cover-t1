# #81 — v3 curl renderer comparison and shared model

## Problem

v3 needs a natural page curl, but implementation should not begin with a renderer-specific animation that repeats v2's two-phase coupling. CSS strips, Canvas 2D, and WebGL must be evaluated against the same geometry, lighting, interaction, and fallback requirements.

## Approach

1. Added a renderer-neutral curl model at `src/v3/lib/curl-model.ts`.
2. Defined stable progress and geometry samples:
   - clamped `progress` and normalized spine-to-edge coordinate `u`
   - smooth commanded-turn easing
   - free-edge lead that fades to zero at both endpoints
   - radius and lift that peak around the middle
   - symmetric gutter-lighting samples
3. Added focused Vitest coverage for endpoint convergence, free-edge lead, direction mirroring, monotonic easing, and lighting symmetry.
4. Recorded the initial renderer decision in the Korean and English v3 plans.

## Renderer Decision Matrix

| Criterion | T1 CSS segmented | T2 Canvas 2D CPU mesh | T3 WebGL/Three.js |
| --- | --- | --- | --- |
| Natural curvature ceiling | Medium | Medium | High |
| Direct progress/drag fit | Good | Good | Good |
| Normal-based lighting | Limited | Manual/expensive | Strong |
| Low-capability fallback | Strong | Weak | Weak |
| Maintenance cost | Low-medium | Medium-high | High |
| Recommended role | Fallback | Deferred | Default quality path |

## Decision

- Use T3 WebGL/Three.js as the target v3 renderer.
- Keep T1 CSS segmented curl as the fallback and reduced-capability path.
- Defer T2 Canvas 2D because it does not provide enough quality or fallback advantage to justify another renderer.
- Revalidate the decision with browser traces and fixed-progress visual comparisons during M3.2c.

## Trade-offs

- The shared model establishes deterministic invariants but does not yet render a visible curl.
- Three.js increases bundle and GPU lifecycle complexity, so route-level lazy loading and explicit disposal remain mandatory.
- CSS fallback will have a lower quality ceiling but preserves availability and reduced-capability behavior.
