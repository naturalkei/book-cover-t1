# #78 — v2 reader static reveal at cover / last spread boundaries

## Problem

After #72 (flicker) and #73 (spine pop), boundary flips still showed the wrong static content beneath the outgoing leaf:

1. **Cover exit** (`coverMode=single`, spread): the left half of the target spread appeared immediately under the animating cover leaf.
2. **Last spread entry**: the last page image (left half) and empty trailing slot (right half) appeared before the flip finished.
3. **Last single page**: the final page image appeared under the outgoing leaf during the animation.
4. **Interior flips** had been over-corrected by deferring the entire static layer, so the target page was hidden until flip completion.

Additionally, classic preset still applied `rotateX` / `translateZ` and phase-boundary `boxShadow` transitions that contributed to Y-axis bobbing and gutter shadow instability.

## Root cause

- `displayIndex` was either updated immediately for all flips (boundary peek) or deferred globally (interior target hidden).
- Spread static surface always rendered `pages[displayIndex]` and `pages[displayIndex + 1]` with no per-half masking at book edges.
- Classic leaf transforms mixed perspective/rotateX on the leaf instead of the container.

## Approach

1. **Immediate target index for most flips** — on `pageIndex` change, set `displayIndex = safeIndex` unless single-mode forward to the last page.
2. **`resolveStaticSpreadMask`** — during an outgoing spread flip:
   - `holdLeft` + `holdLeftBlank`: leaving cover alone page → blank left half.
   - `holdLeft` + stale outgoing left: forward onto last spread (no trailing page) → keep previous spread left.
   - `holdRight`: forward onto last spread → keep previous spread right.
3. **Single last page defer** — keep `displayIndex` on penultimate page until `completeFlip`, then sync to `safeIndex`.
4. **Classic preset cleanup** — `rotateY` only on leaves; `FLIP_PERSPECTIVE` on the flip container; constant `boxShadow` in spread mode; spine overlay at fixed opacity (no phase-sync ramp).

v1 reader untouched.

## Trade-offs

- Last spread / last single page static content lags one animation behind the logical `pageIndex`; acceptable because the outgoing leaf carries the visible motion.
- Spread gutter depth is slightly flatter (constant leaf shadow) but more stable than phase-boundary shadow steps.
- Single-mode interior flips show the target immediately under the leaf (intentional — fixes hidden-target regression).
