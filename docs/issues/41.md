# Issue #41 — feat(data): expand books to ≥ 20 pages with unique per-page content

- **Status**: Resolved
- **Milestone**: M6 — Reader Enhancements
- **Branch**: `feat/41-rich-book-pages`

## Motivation

Every book in `src/data/books.ts` previously held 8–12 copies of the same `page-blank.svg`, which made the reader feel like a slideshow of identical paper:

- Page flips never revealed new content — the animation, even when working, looked empty.
- Spread mode was indistinguishable from single mode.
- The thumbnail scrubber was a flat strip of clones, contradicting its own purpose.
- The lazy preload window (`±2`) and the per-image cache hooks had nothing to prove.

## Resolution

### 1. Programmatic page generator — `src/data/bookPages.ts`

A small module that turns book metadata into rich, per-page SVG data URIs:

```ts
createBookPages({ bookId, title, author, accentColor, count }): string[]
```

The generator plans the page sequence then renders each page as a compact SVG that is URL-encoded into a `data:image/svg+xml;charset=utf-8,...` URI — no extra static files, no extra network requests.

### 2. Page schedule

For a book of `count` pages:

| Index             | Layout      | Body                                                                                                  |
| ----------------- | ----------- | ----------------------------------------------------------------------------------------------------- |
| `0`               | `cover`     | Large title, uppercased author, accent rule, "VOLUME ONE" mark.                                       |
| every 5th         | `figure`    | One of five geometric studies (`concentric`, `columns`, `horizon`, `arc`, `grid`) themed by accent.   |
| `count - 1`       | `colophon`  | "— colophon —" mark, byline, set-in note.                                                             |
| all others        | `text`      | A typography page drawn from a pool of 12 short lorem blocks, varied per page index.                  |

Every layout also paints:
- A subtle paper-grain background (cream `#faf6ee` flecked with `#e7dccb` dots).
- An accent header band tinted by the book's `accentColor`.
- A footer with `{title} · {author}` on the left and `{n} / {total}` on the right.

### 3. Per-book counts (≥ 20)

| Book                  | Pages |
| --------------------- | ----- |
| Atlas of Cities       | 24    |
| Silent Frequencies    | 28    |
| Coastal Tales         | 22    |

### 4. Why data URIs over static SVG files

- **Single source of truth** — change the page template once, ripple through 74 pages.
- **Zero new HTTP requests** — the existing perf E2E expectations (lazy module fetch, single DOM image at rest, eager `fetchpriority`) all still hold because the URI is inlined into `<img src>`.
- **Trivial uniqueness guarantee** — the generator embeds title/author/index/layout/figure-kind into each URI so collisions are structurally impossible.

### 5. Accent colour, safely

`softenAccent(hex)` blends each book's accent 55 % toward white to produce a sibling colour for figure backgrounds and dividers. This keeps figure pages visibly themed without being garish.

### 6. XML-safe title/author injection

`escapeText()` HTML-encodes `&<>"` before interpolation. The uppercase author on the cover is uppercased **before** escaping — applying `.toUpperCase()` after escaping would have corrupted the entity letters (`&QUOT;` instead of `&quot;`).

## Test Updates

### Unit (`pnpm test`) — 17 files, **138 tests passing**

- **New** `src/data/bookPages.test.ts` — 9 specs covering count, uniqueness, data-URI format, cover/colophon content, page numbering, figure presence, accent threading, and XSS-style escaping.
- **Updated** `src/data/books.test.ts` — now asserts `≥ 20 pages`, per-book page uniqueness, and the `data:image/svg+xml` page format (cover still under `/books/*.svg`).
- **Updated** `src/pages/Reader.test.tsx` — pagination assertion relaxed from hard-coded `1/8` to `1/<two-or-more-digit total>`.

### E2E (`pnpm e2e`) — **27 specs passing** (+2 new)

- **New** `e2e/reader-content.spec.ts`
  - `the first book reports at least 20 pages in the gallery card and reader pagination` — verifies the `BookCard` "N pages" badge and the reader's `reader-total-pages` testid both show ≥ 20.
  - `successive pages render distinct page images (no repeated placeholder)` — clicks Next five times and asserts five distinct `data:image/svg+xml` srcs.

## Bundle Impact

- `dist/assets/index-*.js` — **77.98 kB → 81.76 kB gz (+3.78 kB)**. Well under the +15 kB ceiling agreed in the issue.
- `dist/assets/Reader-*.js` — **6.57 kB gz (unchanged)**. The generator lives in the data layer and is consumed at module init, not in the Reader route chunk.
- No new CSS.

The data URIs themselves are computed at module load and live only in memory — they never appear as string literals in the bundle.

## Follow-ups

- The lorem pool is in English; if M7 introduces localisation, lift it to a per-locale resource.
- The figure pool currently caps at 5 kinds — easy to extend by appending to `FIGURES` and `buildFigure()`.
- `public/books/page-blank.svg` is now unreferenced — left in place as a gallery fallback for any future book that ships before its pages do.
