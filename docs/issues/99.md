# Issue #99: Remove the final v3 page handoff flicker

## Problem

The v3 reader committed the target static page, removed the fully turned leaf,
and reset progress in one React update. That single-paint compositing handoff
could briefly expose an unpainted or differently composited frame.

## Approach

- Add an explicit `handoff` phase after the flip reaches progress `1`.
- Commit the target static page beneath the still-mounted final leaf.
- Remove the leaf and reset progress on the following animation frame.
- Cancel pending handoff frames when a transition is superseded, motion
  preferences change, or the component unmounts.
- Keep reduced-motion cleanup synchronous.

## Trade-offs

- The reader remains busy for one additional animation frame, normally about
  16 milliseconds, to guarantee overlap between the final leaf and target
  static page.
