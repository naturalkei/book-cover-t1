# #82 — v3 routes and reader shell

## Problem

v3 planning (#80, plan-3) defined a progress-driven curl reader, but the app had no `/v3` routes, gallery, or reader entry point. Development could not proceed in isolation from v1/v2.

## Approach

1. Added `@v3/*` path alias in `tsconfig.app.json`.
2. Scaffolded `src/v3/` with layout, gallery, NotFound, books data (forked, no v2 imports), and reader stub.
3. Wired `/v3` and `/v3/book/:id` in `App.tsx` with lazy-loaded Reader.
4. Added v3 card to VersionHub and v3 link on root 404.
5. Added `e2e/v3/smoke.spec.ts` and extended version-hub e2e.

v1 and v2 untouched.

## Trade-offs

- Reader is a stub until #83 lands the flip engine.
- Book data and page generator are duplicated from v2 per version isolation rules.
