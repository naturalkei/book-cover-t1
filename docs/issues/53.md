# Issue #53 — Tap left half of first spread to return to the cover

## Symptom

In spread mode with `coverMode='single'`, advancing from the cover (`pageIndex
= 0`) lands the user on the first interior spread (`pageIndex = 1`). From
there, clicking the **left half** of the page surface did nothing. The Prev
button and ArrowLeft keyboard shortcut both already returned to the cover, so
only the tap-to-navigate path was missing the affordance.

## Root Cause

`PageFlip.handleClick` computed `next = safeIndex - step`. At `pageIndex = 1`
with cover-aware `step = 2`, `next = -1`, and the handler bailed via
`if (next < 0) return`. That blanket guard ignored the fact that a cover page
exists at `0`.

## Resolution

Replaced the underflow guard with a cover-aware clamp:

```ts
let next = rawNext
if (next < 0) {
  if (safeIndex === 0) return
  next = 0
}
if (next >= pages.length) return
onPageChange?.(next)
```

Behavior:

- `pageIndex = 1`, `step = 2` (spread + cover-alone first spread): `next = -1`
  → clamped to `0` → reader returns to the cover.
- `pageIndex = 0` (cover-alone, or first page of single mode, or first spread
  of spread + `coverMode='spread'`): `next < 0` but `safeIndex === 0` → bail.
  No regression — we still cannot navigate before the start of the book.
- Every other case lands on a non-negative step and is unaffected.

The Prev button and the ArrowLeft keyboard shortcut were already routing
through `commitPage` (which clamps via `Math.max(0, next)` before snapping),
so they continue to work as before.

## Regression Coverage

- `src/components/reader/PageFlip.test.tsx`:
  - clicking the left half of the first spread snaps back to the cover
  - clicking the left half at the cover itself still does nothing
- `e2e/reader-cover-mode.spec.ts`:
  - tapping the left half of the first spread closes back to the cover
