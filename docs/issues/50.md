# Issue #50 — Tilt preset shrink-then-grow glitch in spread mode

## Symptom

With the **tilt** preset selected, every spread-mode flip:

1. Starts with the leaf at full size.
2. Visibly **shrinks** during the rotation.
3. Snaps back to **full size** the instant the leaf is removed and the static
   spread takes over.

The effect reads as a pulse and breaks the "real book" illusion the preset is
supposed to convey.

## Root Cause

The tilt preset's spread-mode final frame applied `scale(0.96)`:

```ts
'perspective(1800px) rotateX(-6deg) rotateY(±178deg) scale(0.96)'
```

The CSS `transform` transition interpolated from `transform: none` (scale 1)
to scale 0.96 over the full animation duration. At cleanup the leaf is removed
and the static spread underneath is at scale 1 — so the user saw the
0.96 → 1.0 jump as a sudden enlargement.

The single-mode tilt deliberately uses `scale(0.92)` because the leaf is
fading out (opacity 1 → 0); in that context the shrink reads as "lifting off"
and the opacity change masks the cleanup boundary. Spread-mode tilt does **not**
fade the leaf — opacity stays at 1 throughout so the back face from #45 can
paint — so the same scale change reads as a glitch.

## Fix

In `src/components/reader/flipPresets.ts`, the tilt preset's spread-mode final
transform drops `scale(...)` entirely:

```ts
mode === 'spread'
  ? 'perspective(1800px) rotateX(-6deg) rotateY(±178deg)'
  : 'perspective(1800px) rotateX(-10deg) rotateY(±150deg) scale(0.92)'
```

The tilt feel (`rotateX(-6deg)`) is preserved. Single-mode tilt keeps its
`scale(0.92)` lift-off effect because its fade-out context still motivates it.

Classic and curl spread-mode presets already had no scale, so they didn't need
to change — but `flipPresets.test.ts` now pins this with a regression
assertion to prevent future preset additions from re-introducing the pulse.

## Regression Coverage

- `src/components/reader/flipPresets.test.ts`:
  - "spread-mode tilt does not scale the leaf"
  - "single-mode tilt keeps its scale(0.92) lift-off effect"
  - "spread-mode classic and curl also keep scale unchanged so the leaf does
    not pulse"
- `e2e/reader-spread-flip.spec.ts`:
  - "tilt preset does not pulse — neither the initial nor the final transform
    applies a scale in spread mode" — asserts the leaf's inline `transform`
    contains `rotateX(-6deg)` but no `scale(`.
