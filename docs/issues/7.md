# Issue #7 — feat(reader): add reader route shell at /book/:id

- **Status**: Resolved
- **Milestone**: M2 — Reader MVP
- **Branch**: `feat/7-reader-route-shell`

## Problem

Gallery cards linked to `/book/:id` but the route did not exist. Following milestones (#8 flip animation, #9 prev/next controls) need a shell to attach to.

## Approach

- Added `src/pages/Reader.tsx`:
  - Reads `:id` via `useParams`, looks up the book through `getBookById`.
  - Renders a header with a "Back to gallery" link, the book title, and the author.
  - Renders a static spread placeholder (the book's first page asset) inside a styled card so the visual rhythm matches what the flip animation will land into.
  - Footer shows `Page N / Total` and an honest disclaimer that flip controls land in #8 / M3.
- Added `src/components/NotFound.tsx`, a shared 404 component that App's catch-all route and Reader's "book not found" branch both consume. It includes a "Back to gallery" link.
- Wired `App.tsx`:
  - Imports `Reader` and `NotFound`.
  - Adds `<Route path="book/:id" element={<Reader />} />` nested under the `Layout`.
  - Replaces the inline 404 markup with `<NotFound />`.

## Trade-offs

- **State deferred to #8**: kept Reader stateless (hard-codes `currentPage = 0`) instead of dragging `useState` in now. Avoids a lint warning for an unused setter, and #8 will introduce real navigation state alongside the flip animation.
- **Inline NotFound vs redirect**: rendering NotFound inline on unknown ids preserves the URL (`/book/foo` stays in the address bar) and avoids history-stack noise. A redirect would have hidden the typo from the user.
- **Spread placeholder uses page 1**: not a true two-page spread — #8 will introduce the spread renderer with shadow and spine. Using page 1 keeps this shell visually credible without anticipating the M2 design.
