# Issue #61 — File naming conventions

## Problem

Source files used mixed naming styles (`bookPages.ts`, `useCoverMode.ts`, `flipPresets.ts`) with no documented convention. This made imports inconsistent and harder for agents/contributors to guess filenames.

## Approach

1. Documented rules in `docs/plan-1.md` §4 and `.cursor/rules/code-style.mdc`:
   - Component `.tsx` files → **PascalCase**
   - All other files and folders → **kebab-case**
   - `src/hooks/` modules → kebab-case **without** `use` prefix in the filename; exported hooks keep `use` (e.g. `cover-mode.ts` → `useCoverMode`)

2. Renamed 17 files via `git mv` (preserves history):
   - `src/data/bookPages.*` → `book-pages.*`
   - `src/components/reader/flipPresets.*` → `flip-presets.*`
   - All `src/hooks/use*.ts(x)` → kebab-case without `use` prefix

3. Updated every import path across components, pages, hooks, and tests.

## Trade-offs

- **No export renames** — only filenames changed; public hook/API names stay `useCoverMode`, etc., so component code changes are limited to import paths.
- **Historical docs** (`docs/issues/*.md`) still reference old paths; left unchanged as archival records.
