# #72 — v2 reader page image flicker during flip

## Problem

During v2 classic page turns, the static page could flash briefly mid-animation. Repro: open `/v2/book/:id`, flip forward/back with DevTools slowed 6× — the incoming page sometimes appeared under the outgoing leaf before the animation finished.

## Root causes (verified)

1. **`PageSurface` remount** — single-mode `<img key={page-${index}}>` forced a new DOM node on every index change, triggering decode/repaint while the outgoing leaf was still visible.
2. **Timeout vs CSS drift** — `setTimeout(..., duration)` cleared `outgoing` independently of `transitionend`, so teardown could lead or lag the visual transition.
3. **Classic single `opacity: 0` at `final`** — the outgoing leaf faded out before unmount, revealing the static layer underneath early (often already showing the *new* page).
4. **Static layer updated immediately** — `PageSurface` bound to `pageIndex` while outgoing still carried the previous page, so two different images competed in the same stack.

## Approach

- **Deferred `displayIndex`** — static surface keeps the outgoing page until flip completion; `safeIndex` still drives aria label, cover-alone flags, and leaf `toIndex`.
- **`useFlipTransitionEnd`** — `transitionend` on the animated node with a `duration + 80ms` fallback (works in jsdom with fake timers).
- **Remove single-mode `key`** — update `src` in place on the same `<img>`.
- **Classic/curl single final opacity → 1** — rely on `backfaceVisibility: hidden` at ±180° instead of an opacity crossfade.
- **Z-index layering** — static `z-0`, phantom `z-[5]`, outgoing `z-10`.

v1 reader untouched (issue scoped to `src/v2/`).

## Trade-offs

- Static page content lags `pageIndex` by one animation — acceptable because outgoing/phantom layers carry the motion; aria label reflects the logical page immediately.
- Opacity-based presets (slide/fade/tilt) still fade to 0; deferred `displayIndex` reduces handoff flash for those too.
- Spine shadow pop at `final` (#73) remains a separate follow-up.
