# Issue #67 — plan-2 Phase 1: v1 directory split and route prefix

- **Status**: Ready for review
- **Branch**: `refactor/67-plan2-phase1`

## Problem

The codebase used a flat `src/` layout with routes at `/` and `/book/:id`. [plan-2](../plan-2.md) requires coexisting v1/v2 trees, a version hub, and `/v1/*` paths while preserving legacy deep links.

## Approach

1. **`git mv` into `src/v1/`** — pages, components, hooks, data, lib; `Layout.tsx` → `layouts/V1Layout.tsx`.
2. **App shell (`src/app/`)** — `App.tsx` with VersionHub at `/`, nested `/v1` and `/v2` routes, legacy `/book/:id` → `/v1/book/:id` redirect.
3. **v2 scaffold (`src/v2/`)** — minimal layout, gallery stub, reader stub (Phase 2 prep).
4. **Path aliases** — `@app/*`, `@v1/*`, `@v2/*`, `@shared/*` in `tsconfig.app.json`.
5. **E2E restructure** — `e2e/v1/`, `e2e/shared/helpers.ts`, `e2e/version-hub.spec.ts`, `e2e/v2/smoke.spec.ts`; gallery paths → `/v1`.
6. **Playwright** — webServer forces `VITE_BASE_URL=/` so local `.env` base path does not break e2e.

## Trade-offs

- **VersionHub + v2 stub in Phase 1 PR** — plan-2 lists VersionHub in Phase 2, but shipping the route shell early avoids a broken `/` and lets CI assert hub + v2 smoke in one pass.
- **v2 does not import v1** — v2 reader is a placeholder, not a re-export, per import isolation rules.
- **RootLayout is a thin outlet** — skip link + `#main-content` live in version-specific layouts to avoid duplicate IDs.
