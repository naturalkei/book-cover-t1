# Issue #69 — GitHub Pages SPA fallback for deep links

- **Status**: Ready for review
- **Branch**: `fix/69-spa-fallback`

## Problem

GitHub Pages returns a server 404 for direct URLs such as `/v1`, `/v2`, and `/book/:id`. The SPA only loaded from `/` because `index.html` was the sole HTML entry.

## Approach

- Add a Vite build plugin (`scripts/spa-fallback.ts`) that copies `dist/index.html` → `dist/404.html` in `closeBundle`.
- GitHub Pages serves `404.html` for unknown paths, bootstrapping React Router with the correct client-side route.

## Trade-offs

- Both files are identical duplicates (~1 KB). Acceptable for static hosting; no server rewrite rules needed.
- Plugin runs on every production build (CI deploy included).

## Verification

- `pnpm build` — `dist/404.html` present
