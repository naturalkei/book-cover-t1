# Issue #44 — feat(reader): rounded-corners toggle (real-book / soft-book look)

- **Status**: Resolved
- **Milestone**: M6 — Reader Enhancements
- **Branch**: `feat/44-rounded-toggle`

## Why
Real books have sharp corners. The reader's previously hard-coded `rounded-2xl` made pages look like UI cards. Add a labelled checkbox to choose between **sharp** (default) and **rounded** corners; persist the choice.

## How
1. **`useRoundedCorners` hook** — `localStorage` key `book-flip-showcase:rounded-corners`, default `false`.
2. **`RoundedToggle` component** — a small `role="switch"` checkbox in the reader header, mirroring the look of the cover-mode and view-mode controls.
3. **`PageFlip`** takes a `rounded` prop and threads a `roundClass` ("" | "rounded-2xl") into:
   - the outer board,
   - each `PageSurface` variant (single, cover-alone, spread),
   - the `OutgoingLayer` (single, cover-alone, spread).
   It exposes `data-rounded` on the surface for tests.
4. Thumbnails keep their tiny `rounded-md` — that's UI chrome, not book paper.

## Acceptance Checks
- [x] Default is `false` (sharp) — matches the real-book brief.
- [x] Toggle switches state immediately and writes `'true'` / `'false'` to `localStorage`.
- [x] When false: no `rounded-2xl` on the board or any page surface (board / outgoing leaf / cover-alone / spread).
- [x] When true: `rounded-2xl` appears on the board, current page surface, outgoing leaf, etc.
- [x] Setting persists across reload.
- [x] Toggle is keyboard-accessible (`role="switch"`).

## Trade-offs
- The toggle reflects state with a `role="switch"` checkbox rather than a segmented control because it is binary. The same pattern is used by `CoverModeToggle`.
- The `roundClass` indirection is verbose at every render site but keeps the corner-policy uniform across all four surface variants. A future refactor could push this into a CSS variable on the board (`--book-radius`) and let descendants inherit.
