# Issue #101: Mark v3 as paused work in progress

## Context

The current v3 prototype does not meet the interaction quality and visual
finish established by v2. Further v3 feature work and release activity are
paused while the prototype remains available for comparison and research.

## Approach

- Add an accessible persistent status badge to the shared v3 layout.
- Show `Work in progress · Paused` on every v3 route.
- Record the pause decision and v2 quality baseline in both v3 plan documents.
- Notify the active v3 roadmap issues without closing them, preserving a clear
  path for a future resumption decision.

## Trade-offs

- The v3 prototype remains navigable instead of being removed, so users can
  still encounter incomplete interactions. The persistent badge makes that
  status explicit.
- Open v3 roadmap issues remain intentionally unresolved while development is
  paused.
