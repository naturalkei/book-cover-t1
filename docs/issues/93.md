# Issue #93: Keep the v3 cover to one page and smooth gutter lighting

## Problem

The v3 reader rendered `coverMode="single"` across the complete two-page
spread. Cover transitions also used a full-width leaf. The gutter lighting
model existed, but the reader did not render it, leaving a single turning-leaf
shadow to represent all overlap lighting.

## Approach

- Render the resting cover in the right-hand half of the spread with a blank
  left slot, matching the established v2 behavior.
- Use a half-width leaf for every spread transition, including transitions to
  and from the cover.
- Add a progress-driven gutter layer with separate crease, cast-shadow, and
  ridge-highlight elements.
- Keep the gutter layer above static pages and below the turning leaf so the
  shadow reads as depth beneath the paper.

## Trade-offs

- The CSS renderer can smoothly approximate gutter lighting, but it cannot
  produce a truly curved paper surface. That remains part of the WebGL work in
  #85.
- The cover transition is now spatially correct, while full rigid-cover
  material behavior remains a later polish item.
